# 片区计划达成情况报表

## 概述

本SQL脚本用于生成各片区销售计划和需求计划的达成情况报表，包含按组织层级和产品维度的执行率统计。

## 功能说明

### 主要指标

| 字段名称 | 说明 | 单位 |
|---------|------|------|
| 片区 | 一级组织区域名称 | - |
| 销售计划KL | 月度销售计划量 | 千升 |
| 需求计划KL | 月度需求计划量 | 千升 |
| 实际KL | 实际销售量 | 千升 |
| 销售计划执行 | 整体销售计划达成率 | 百分比 |
| 需求计划执行 | 整体需求计划达成率 | 百分比 |
| 重点产品1 | 重点高端产品销售计划达成率 | 百分比 |
| 常规产品1 | 常规产品销售计划达成率 | 百分比 |
| 小计1 | 所有产品销售计划达成率汇总 | 百分比 |
| 重点产品2 | 重点高端产品需求计划达成率 | 百分比 |
| 常规产品2 | 常规产品需求计划达成率 | 百分比 |
| 小计2 | 所有产品需求计划达成率汇总 | 百分比 |
| 目标准确率 | 预设目标准确率 | 80% |
| 与目标比差异 | 实际准确率与目标的差值 | 百分比 |

## 数据来源

### 数据表
1. **`crb_edw_dws_op.dws_op_oc_three_month_plan_agg_mf`** - 三个月计划聚合表
   - 提供销售计划和需求计划数据
   - 按组织层级和产品维度聚合

2. **`crb_edw_dws_op.dws_op_sales_bill_agg_df`** - 销售单据聚合表
   - 提供实际销售数据
   - 按月度维度统计

## 计算逻辑

### 计划执行率公式
```
执行率 = (1 - |计划量 - 实际量| / 计划量) × 100%
```

### 产品分类
- **重点高端产品**：`op_second_brand_name = '重点高端'` 标记为 1
- **常规产品**：其他产品标记为 0

## 统计维度

1. **组织维度**：按片区（一级组织）汇总
2. **时间维度**：
   - 计划月份：报表月份的上一个月
   - 统计月份：2025年4月
3. **产品维度**：区分重点高端产品和常规产品

## 使用说明

### 环境要求
- Hive / Spark SQL 执行环境
- 数据仓库权限访问

### 修改参数
如需修改统计月份，请调整以下两处：

1. 计划数据过滤条件（第36行）：
```sql
where report_month = DATE_FORMAT(DATE_SUB(STR_TO_DATE(CONCAT('2025-04', '-01'), '%Y-%m-%d'), INTERVAL 1 MONTH), '%Y-%m')
```

2. 实际销售数据过滤条件（第45行、第82行）：
```sql
and line_bill_created_date = '2025-04-01'
```

## 输出结果

查询结果按片区名称排序，可直接导出为Excel用于月度经营分析会议。
